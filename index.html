Github Flow

Github Flow är ett arbetssätt man bör hålla sig till då man arbetar i samband med andra utvecklare. De grunläggande pinciperna är att man jobbar inom en egen branch som sedan går igenom diverse checkar för att sedan "merga" med ett större project. Genom att hålla sig till ett facit för facit för att förminska chanserna att något går fel i processen av att samansätta den koden man gemansamt jobbar på.
Utan flödet löper man risk att kliva på någon tår eller helt och hållet ersätta deras kod med ens egen om man får för sig att pusha och merga något utan följa det arbetssätt du och dina medarbetare kommit överns över.

Flexbox

Flexbox är ett sätt att designa hemsidor med hjälp utav CSS. Just Flexbox har en markant fördel i och med det att göra hemsidor dynamiska och anpassade för alla tänkbara skärmstorlekar. Raderna eller kollumnerna och dess inehåll är flexibla på det sättet att de kan sjusterar sin ordning och storlek beroende på skärmstorleken. Denna hemsidan är ett exempel på hur Flexbox kan se ut. Testa att försora och förminska hemsidan för att se hur de olika textfälten anpassar sig.

Fin Kod

På samma sätt som Github Flow är fördelsfullt när man jobbar med an dra, är fin kod minst lika viktig. Fin kod innebär enklast sagt att den är lättläst. Det kan vara allt ifrån hur HTML element är indenterade till det så kallade camelCase man använder sig utav inom JavaScript och många andra programmeringsspråk. För att andra utvecklare ska kunna läsa, förstå och arbeta vidare med din kod är det viktigt att ha fin kod i åtanke.

Ramverk

Ett ramverk är en sorts resurs som går t.ex byggnad av hemsider enklare. Bland annat Bootstrap tillåter dig att bygga din hemside med hjälp utav fädriga CSS klasser vilket tillåter en programmerare att spara tid och jobba mer effektivt.
Ett annat ramverk är Angular. Liknande Bootstrap så tillåter det dig att smidigare skriva markup. Till skillnad från Bootstrap är det inte ett CSS ramverk men ger istället tillgång till en bred mängder möjligheter inom JavaScript.
